# `unused_function`

`unused_function` flags function-shaped declarations that are never referenced
across the analyzed file set.

| Field | Value |
| ----- | ----- |
| Rule id | `unused_function` |
| Default severity | `warning` |
| Dispatch | multi-file |

## What it reports

The rule reports unreferenced:

- private top-level functions
- public top-level functions under `lib/src/`
- top-level getters and setters, using the same public/private policy as
  top-level functions
- local function declarations
- constructors on classes and enums
- instance methods
- static methods
- getters
- setters
- operators
- extension members

Diagnostics use this shape:

```text
The <kind> "<name>" is declared but never used.
```

The correction is:

```text
Remove "<name>" or reference it.
```

## What counts as a use

The rule builds a global reference set from every resolved compilation unit in
the run, including excluded files. These count as references:

- direct calls
- tear-offs
- constructor invocations
- redirecting constructors
- implicit super-constructor calls
- enum-value constructor calls
- member reads and writes
- static member access
- operator invocations
- cascades
- callable-object invocations through implicit `.call`
- object-pattern and record-pattern reads
- record literals
- generic member calls after normalization to the declared base element
- references from excluded files such as generated `*.g.dart` files

## Public API policy

The rule avoids flagging public API that external consumers may call even when
the analyzed project does not reference it internally.

It does not report:

- public top-level declarations outside `lib/src/`
- public members of public classes, mixins, enums, extension types, and
  extensions outside `lib/src/`

Inside `lib/src/`, public top-level declarations are treated as internal
implementation details and may be reported.

## Important exemptions

The rule deliberately skips:

- `main`
- `external` declarations
- declarations annotated with `@pragma('vm:entry-point')`
- members and constructors in units that import `dart:mirrors`
- all candidates in files starting with `// ignore_for_file: type=lint`
- members and constructors whose supertype chain uses `noSuchMethod`
- members of `Fake` or `Mock` style test doubles
- overrides of reachable supertype members, with or without `@override`
- declarations in non-selected conditional import/export branch files
- constructors of classes annotated with Freezed-style annotations
- members of private, unreferenced enclosing types
- declarations in files already reported by `unused_source_file`

## Conditional imports and exports

A conditional URI such as:

```dart
export 'mobile.dart' if (dart.library.html) 'web.dart';
```

resolves to one branch for the current analysis platform. LintForge still treats
every configured branch as reachable. This prevents false positives in branch
files that are part of the package surface for a different target.

## Generated code

Generated files are usually excluded by default, but they still contribute
references. The rule also skips every candidate in a unit stamped with:

```dart
// ignore_for_file: type=lint
```

Flutter localization output generated by `flutter gen-l10n` uses this marker,
so localization getter surfaces are not reported as unused.

## Examples

Flagged:

```dart
void _unusedHelper() {}

class Service {
  void _unusedMethod() {}
}
```

Not flagged:

```dart
void _usedHelper() {}

void main() {
  final helper = _usedHelper;
  helper();
}
```

Public top-level declarations under `lib/src/` are candidates:

```dart
// lib/src/internal_api.dart
void unusedInternalHelper() {}
```

Public top-level declarations directly under `lib/` are package surface and are
not candidates:

```dart
// lib/my_package.dart
void publicApi() {}
```
